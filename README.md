# 2025 웹프로그래밍 프로젝트
> 2025.10.18 ~

### 1️. 기획 및 설계 단계

**목표:** 서비스의 목적, 기능, 흐름, UI 구조를 명확히 정의  
**주요 작업:**
- 서비스 핵심 컨셉 확정: “텍스트 → AI 멜로디 데이터 → 브라우저 재생”
- 주요 기능 정의
    - 텍스트 입력
    - AI 멜로디 생성 (Gemini API 호출)
    - 멜로디 데이터 표시
    - 재생 (Web Audio API 활용)
- UI 와이어프레임 설계
    - 중앙 입력창
    - 생성 버튼 / 상태 메시지 / 결과 영역 / 재생 버튼 배치
- 데이터 구조 설계
    - api 입력: `prompt (string)`
    - api 출력: `melody_data (JSON)` 예: `{ notes: [{ pitch: "C4", duration: 0.5 }, ...] }`

---

### 2️. 환경 설정 및 기본 구조 구현

**목표:** 백엔드 없는 클라이언트 전용 구조를 완성  
**주요 작업:**
- HTML 기본 뼈대 작성
    - `<textarea>`, `<button>`, `<div id="result">`, `<audio>` 영역 구성
- CSS 스타일링
    - 심플한 중앙 정렬, 반응형 디자인, 상태 메시지 시각화
- JavaScript 파일 분리 및 초기 스크립트 구조 설계
    - 이벤트 리스너 (`onclick`) 설정
    - 상태 메시지 출력 함수 구현 (`showStatus(message)`)

---

### 3️. Gemini API 연동

**목표:** 사용자 입력 → Cerebras Cloud API 요청 → 멜로디 데이터 수신
**주요 작업:**
- Cerebras Cloud API(model: gpt-oss-120b) 키 설정 및 요청 함수 구현
- 요청 전/후 상태 표시 (`"생성 중..."`, `"완료"` 등)
- 오류 처리 및 사용자 알림 구현
---
### 4️. 멜로디 데이터 파싱 및 시각화

**목표:** AI가 생성한 멜로디 데이터를 화면에 표현  
**주요 작업:**

- 수신한 JSON 데이터 파싱
- 결과를 텍스트 혹은 간단한 악보 형태로 표시
    - 예: 음표명, 길이 리스트
    - 추후 Canvas 또는 SVG로 악보 렌더링 고려
- 재생 버튼 활성화 및 데이터 전달 준비
---
### 5️. 브라우저 재생 기능 구현

**목표:** 수신된 음표 시퀀스를 실제 사운드로 재생  
**주요 작업:**
- 웹 Audio API로 간단한 신시사이저 구현
- ex\) `getFrequency(pitch)` 함수 작성 (C4=261.63Hz 등)
- 재생 중 상태 메시지 표시 (“재생 중... / 완료”)
---
### 6️. 테스트 및 UX 개선

**목표:** 전반적인 사용자 경험 개선 및 안정화  
**주요 작업:**
- 다양한 입력 문장 테스트 (“밝은 피아노 멜로디”, “신나는 8비트 게임음”)
- 예외 처리 보강 (API 실패, 응답 지연, 잘못된 데이터 등)
- 시각적 피드백 추가
    - 로딩 애니메이션
    - 결과 전환 효과 (Fade-in)
- 브라우저 호환성 확인 (Chrome / Edge / Safari)
- 최종 UI 다듬기 및 설명 문구 추가 (“입력 후 생성 버튼을 눌러보세요!”)
---

**최종 산출물:**
- 하나의 HTML 파일로 완성되는 클라이언트 전용 AI 멜로디 생성 웹앱
- 사용자는 단순 텍스트 입력만으로 즉시 멜로디를 생성하고 재생할 수 있음
